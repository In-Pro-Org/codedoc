linkPSWriteHTML - Diagram Events

One of the new features I've worked on was connecting Diagrams with Tables. Someone suggested, and I thought it would be cool to be able to click on the Diagram node and find more details about it in a table next to it. But then I thought it would be even cooler if you could have multiple tables linked to one Diagram. For example, below, I've created two tables with Users and Computers and populated Diagram with that data.

I've also added Calendar (oh right, did I mention it's available now?) to be able to show some events on it if needed.

1link$Computers = Get-ADComputer -Filter * -Properties OperatingSystem, OperatingSystemVersion, PasswordLastSet | Select-Object -Property DNSHostName, Name, Enabled, OperatingSystem, OperatingSystemVersion, PasswordLastSet

2link$Users = Get-ADUser -Filter * -Properties SamAccountName, Name, Title, Surname, PasswordNeverExpires, WhenCreated, WhenChanged | Select-Object UserPrincipalName, SamAccountName, Name, Title, Surname, PasswordNeverExpires, WhenCreated, WhenChanged #-First 17

3link

4linkNew-HTML -TitleText 'My Title' -UseCssLinks:$true -UseJavaScriptLinks:$true -FilePath $PSScriptRootExample30-LinkedTable.html -ShowHTML {

5link    #New-HTMLLogo

6link    New-HTMLTabOptions -SelectorColor Grey

7link    New-HTMLTab -TabName 'Test' -IconSolid dice {

8link        New-HTMLContent -HeaderText 'Active Directory Computers & Diagram' -HeaderBackGroundColor Grey {

9link            New-HTMLPanel {

10link                New-HTMLTable -DataTable $Computers -DataTableID 'OtherTable'

11link            }

12link            New-HTMLPanel {

13link                New-HTMLDiagram {

14link                    New-DiagramNode -Label 'Domain' -To 'AD Computers', 'AD Users' -IconBrands delicious

15link                    New-DiagramNode -Label 'AD Computers' -IconBrands apple

16link                    New-DiagramNode -Label 'AD Users' -IconBrands instagram

17link                    New-DiagramEvent -ID 'SpecialID123' -ColumnID 1 #5-FadeSearch

20link                        if ($_.OperatingSystem -like '*Server*') {

21link                            New-DiagramNode -Label $_.Name -IconSolid server -To 'AD Computers'

26link                    }

28link                        New-DiagramNode -Label $_.SamAccountName -To 'AD Users' -IconRegular user-circle

29link                    }

30link                }

31link            }

32link        }

33link        New-HTMLSection -HeaderText 'Active Directory Users' -HeaderBackGroundColor Grey {

35link                New-HTMLTable -DataTable $Users -DataTableID 'SpecialID123' #-DisablePaging

36link            }

37link            New-HTMLPanel {

While the example above may be a tad complicated, and you may think linking Tables to Diagram is hard, it isn't. Linking two tables on ColumnID is just a matter of two simple lines.

1link

2linkNew-DiagramEvent -ID $DataTableIDUsers -ColumnID 1

3linkNew-DiagramEvent -ID $DataTableIDComputers -ColumnID 1

4link

As you see above, there are two parameters. One is called ID and one ColumnID. The ID is related to table ID, and ColumnID is just a column index. Table ID in standard usage is automatically generated, and you usually shouldn't care about it. But for linking Diagram with Table, it's necessary to be able to tell which table is related to a Diagram as there may be multiple Diagrams and multiple Tables. This means when you define New-HTMLTable, you need to set random DataTableID that you then pass to the New-DiagramEvent ID parameter.

1link$DataTableIDComputers = Get-Random -Minimum 100000 -Maximum 2000000

2linkNew-HTMLTable -DataTable $Computers -DataTableID $DataTableIDComputers

To give you a bit simpler example - this one should show how it works without all the complicated stuff.

1link$Processes = Get-Process | Select-Object -First 10

2link$TableID = 'RandomID'

3link

4linkNew-HTML -TitleText 'My Title' -UseCssLinks -UseJavaScriptLinks -FilePath $PSScriptRoot\Example30-LinkedProcesses.html -ShowHTML {

5link    New-HTMLSection -Invisible {

6link        New-HTMLPanel {

7link            New-HTMLTable -DataTable $Processes -DataTableID $TableID

8link        }

9link        New-HTMLPanel {

10link            New-HTMLDiagram -Height '1000px' {

11link                New-DiagramEvent -ID $TableID -ColumnID 1

12link                New-DiagramNode -Label 'Processes' -IconBrands delicious

13link                foreach ($_ in $Processes) {

14link                    New-DiagramNode -Label $_.ProcessName -Id $_.Id -To 'Processes'

15link                }

16link            }

17link        }

18link    }

19link}

Anytime you click on a node, the table on the left will trigger a match and show your choice.

linkPSWriteHTML - Nested Tabs

To show you nested tabs, I'm going to use another example. It is almost the same code I've used above, but in this case, I've queried two domains that I've within one forest as you can see it created Tabs for each domain separately and then within each domain it created Computers and Users tabs. You could go multilevel deep and split computers into Servers and Desktops.

I've also spiced up diagrams a bit to add some colors used green, yellow, gray for a different type of object on the Diagram.

1link$Forest = Get-ADForest

2linkNew-HTML -TitleText 'My Title' -UseCssLinks:$true -UseJavaScriptLinks:$true -FilePath $PSScriptRootExample30-NestedTabs.html -ShowHTML {

3link    New-HTMLTabOptions -SelectorColor Grey

4link    foreach ($Domain in $Forest.Domains) {

5link        $Computers = Get-ADComputer -Server $Domain -Filter * -Properties OperatingSystem, OperatingSystemVersion, PasswordLastSet | Select-Object -Property DNSHostName, Name, Enabled, OperatingSystem, OperatingSystemVersion, PasswordLastSet

6link        $Users = Get-ADUser -Server $Domain -Filter * -Properties SamAccountName, Name, Title, Surname, PasswordNeverExpires, WhenCreated, WhenChanged | Select-Object UserPrincipalName, SamAccountName, Name, Title, Surname, PasswordNeverExpires, WhenCreated, WhenChanged #-First 17

7link        $DataTableIDUsers = Get-Random -Minimum 100000 -Maximum 2000000

8link        $DataTableIDComputers = Get-Random -Minimum 100000 -Maximum 2000000

9link

10link        New-HTMLTab -TabName "Domain $Domain" -IconSolid dice {

11link            New-HTMLTab -TabName 'Computers' {

12link                New-HTMLContent -HeaderText 'Active Directory Computers & Diagram' -HeaderBackGroundColor Grey {

13link                    New-HTMLPanel {

14link                        New-HTMLTable -DataTable $Computers -DataTableID $DataTableIDComputers

15link                    }

16link                    New-HTMLPanel {

17link                        New-HTMLDiagram {

18link                            New-DiagramNode -Label 'Domain' -To 'AD Computers', 'AD Users' -IconBrands delicious -IconColor Red

19link                            New-DiagramNode -Label 'AD Computers' -IconBrands apple -IconColor Black

20link                            New-DiagramNode -Label 'AD Users' -IconBrands instagram -IconColor Yellow

21link                            New-DiagramEvent -ID $DataTableIDUsers -ColumnID 1

22link                            New-DiagramEvent -ID $DataTableIDComputers -ColumnID 1

23link                            foreach ($_ in $Computers) {

26link                                }

27link                                else {

28link                                    New-DiagramNode -Label $_.Name -IconSolid desktop -To 'AD Computers' -IconColor Gray

29link                                }

30link                            }

32link                                New-DiagramNode -Label $_.SamAccountName -To 'AD Users' -IconRegular user-circle

33link                            }

34link                        }

35link                    }

36link                }

37link            }

39link                New-HTMLSection -HeaderText 'Active Directory Users' -HeaderBackGroundColor Grey {

41link                        New-HTMLTable -DataTable $Users -DataTableID $DataTableIDUsers

44link                        New-HTMLCalendar {

47link                                New-CalendarEvent -StartDate $_.WhenChanged -Title "User: $($_.SamAccountName)" -Description "User $($_.Name) modified on $($_.WhenChanged)"

48link                            }

49link                            foreach ($_ in $Computers) {

50link                                New-CalendarEvent -StartDate $_.PasswordLastSet -Title "Computer $($_.SamAccountName)" -Description "Computer $($_.DNSHostName) password last set $($_.PasswordLastSet)"

linkPSWriteHTML - Multiple Levels of Nested Tabs

Nested tabs don't work with only 1 level. You can have as many levels as you want, making it much easier to separate content. Below most of the tabs are just two levels, but in one case, I've added 4th level.

1link$Test1 = Get-Process | Select-Object -First 5

2link

4link    New-HTMLTabOptions -SlimTabs -Transition -LinearGradient -SelectorColor Gold -SelectorColorTarget AliceBlue

5link    New-HTMLTab -Name 'First Level Tab - Test 1' -IconBrands acquisitions-incorporated {

6link        New-HTMLTab -Name '2nd Level Tab - Test 1/1' -IconRegular address-card {

7link            New-HTMLTab -Name '3rd level' {

8link                New-HTMLTab -Name '4th' {

9link                    New-HTMLTable -ArrayOfObjects $Test1 -HideFooter

10link                }

11link            }

12link        }

13link        New-HTMLTab -Name '2nd Level Tab - Test 1/2' -IconRegular address-card {

14link            New-HTMLTable -ArrayOfObjects $Test1 -HideFooter

15link        }

16link        New-HTMLTab -Name '2nd Level Tab - Test 1/3' -IconRegular address-card {

17link            New-HTMLTable -ArrayOfObjects $Test1 -HideFooter

18link        }

19link        New-HTMLTab -Name '2nd Level Tab - Test 1/4' -IconRegular address-card {

The only caveat here is that you need to make sure to always define New-HTMLTab next to each other. You shouldn't try and add Tabs within sections as it breaks tabs.

linkPSWriteHTML - Diagram Example

As you can notice, nodes were given fixed positions using X and Y parameters. 6link        New-DiagramOptionsNodes -WidthConstraintMaximum 200 -Margin 10 -Shape box

8link

9link        New-DiagramNode -id 100 -Label "This node has no fixed, minimum or maximum width or height" -X -50 -Y -300

11link        New-DiagramNode -id 211 -WidthConstraintMinimum 120 -Label "...so does this" -X -500 -Y -50

linkPSWriteHTML - Calendar Usage

While I've mentioned Calendar before, I think it's a good idea to show you a simple example so you can use it for your own needs.

1link$ProcessSmaller = Get-Process | Select-Object -First 1

2link

3linkNew-HTML -Name 'Test' -FilePath "$PSScriptRootExample28-Calendar01.html" -Show {

4link    New-HTMLSection {

5link        New-HTMLSection -HeaderText 'Test 1' {

6link            New-HTMLTable -DataTable $ProcessSmaller

7link        }

8link        New-HTMLSection -HeaderText 'Test 2' {

9link            New-HTMLCalendar {

10link                New-CalendarEvent -Title 'Active Directory Meeting' -Description 'We will talk about stuff' -StartDate (Get-Date)

11link                New-CalendarEvent -Title 'Lunch' -StartDate (Get-Date).AddDays(2).AddHours(-3) -EndDate (Get-Date).AddDays(3) -Description 'Very long lunch'

12link            }

13link        }

14link    }

15link} -UseCssLinks -UseJavaScriptLinks

As you can see in the above example, adding a new calendar is as easy as using other PSWriteHTML features.

1linkNew-HTMLCalendar {

2link    New-CalendarEvent -Title 'Active Directory Meeting' -Description 'We will talk about stuff' -StartDate (Get-Date)

3link    New-CalendarEvent -Title 'Lunch' -StartDate (Get-Date).AddDays(2).AddHours(-3) -EndDate (Get-Date).AddDays(3) -Description 'Very long lunch'

4link}

Just open up New-HTMLCalendar, and add events using New-CalendarEvent. Define a Title, StartDate, and optionally EndDate. You can also add a Description that will show up when you hover over an event.

Change one line, and you can pick a different direction from left to right, down to up, and vice versa.
