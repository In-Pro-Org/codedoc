When I try to learn a new technical topic it is easier for me to do/experience the new topic through a technology I already know about. That’s why I prefer to play around with things in PowerShell even though there might be better ways of doing the same using another technology.

In this post, I’m going to explore a bit of graph theory based on chapter 2 of the excellent book “Think Complexity 2e” by Allen B. Downey (in fact I’d highly recommend any of the books in the “Think…” series some of which I might cover in future posts). The book and the book’s source (in Python) are available for free through the book’s webpage. Graph theory is the study of graphs, which are mathematical structures used to model pairwise relations between objects. A graph is made up of vertices (also called nodes or points) which are connected by edges (also called links or lines). Since networks are everywhere, graph theory is everywhere, too. Graph theory is used to model and study all kinds of things that affect our daily lives: from traffic routes to social networks or integrated circuits.

linkComplete graphs

A complete graph is a graph where every node is connected to every other. Let’s create a function that draws a complete graph based on a list of Nodes. With this function, we’ll also start returning a Graph object with Nodes, Edges, and Visual properties.

1linkfunction Get-CompleteGraph($Nodes) {

2link    $ht = [ordered]@{}

3link    $ht.Nodes = $Nodes

4link    $ht.Edges = for ($i = 0; $i -lt $Nodes.Count; $i++) {

5link        for ($j = 0; $j -lt $nodes.Count; $j++) {

6link            if ($i -lt $j) {

7link                New-Edge $Nodes[$i] $Nodes[$j] -AsObject

8link            }

9link        }

10link    }

11link    $ht.Visual = graph {

12link        inline 'edge [arrowsize=0]'

13link        edge $ht.Edges -FromScript { $_.To } -ToScript { $_.From }

14link    }

15link    [PSCustomObject]$ht

16link}

linkRandom graphs

A random graph is just what it sounds like: a graph with nodes and edges generated at random.

One of the more interesting kinds of random graphs is the Erdös-Renyi model, studied by Paul Erdös and Alfred Renyi in the 1960s. An Erdös-Renyi graph (ER graph) is characterized by two parameters:

  1. The number of nodes
  2. The probability that there is an edge between any two nodes

Erdös and Renyi studied the properties of these random graphs; one of their surprising results is the existence of abrupt changes in the properties of random graphs as random edges are added. One of the properties that displays this kind of transition is connectivity (An undirected graph is connected if there is a path from every node to every other node.)

In an ER graph, the probability (p) that the graph is connected is very low when p is small and nearly 1 when p is large. Let’s create a function that creates ER random graphs where the probability of an edge between each pair of nodes can be controlled via the Probability parameter.

1linkfunction Get-RandomGraph ($NodeCount, $Probability, [switch]$NoVisual) {

2link    $ht = [ordered]@{}

3link    $ht.Edges = for ($i = 0; $i -le $NodeCount - 1; $i++) {

4link        for ($j = 0; $j -le $NodeCount - 1; $j++) {

5link            if ($i -lt $j) {

6link                $rand = (Get-Random -Minimum 0 -Maximum 10000) / 10000

7link                if ($rand -lt $Probability) {

8link                    New-Edge $i $j -AsObject

9link                }

10link            }

11link        }

12link    }

13link    $ht.Nodes = 0..($NodeCount - 1)

14link    if (-not $NoVisual) {

15link        $ht.Visual = graph -Name Random {

16link            inline 'edge [arrowsize=0]'

17link            node $ht.Nodes

18link            edge $ht.Edges -FromScript { $_.From } -ToScript { $_.To }

19link        }

20link    }

21link    [PSCustomObject]$ht

22link}

Remember? A graph is connected if there is a path from every node to every other node For many applications involving graphs, it is useful to check whether a graph is connected. An algorithm that does this starts at any node and checks whether you can reach all other nodes. If a node can be reached, it implies, that you can also reach any of its neighbour nodes. Below is a function that returns the neighbours of a given node by its name.

1linkfunction Get-Neighbours ($Edges, $Name, [switch]$Undirected) {

2link    $edgeObjects = @($Edges)

3link    if (@($Edges)[0].GetType().FullName -ne 'System.Management.Automation.PSCustomObject') {

4link        $edgeObjects = foreach ($edge in $Edges) {

5link            [PSCustomObject]$edge

6link        }

7link    }

8link    (& {

9link            ($edgeObjects.where{ $_.From -eq $Name }).To

10link            if ($Undirected) {

11link                ($edgeObjects.where{ $_.To -eq $Name }).From

12link            }

13link        }).where{ ![String]::IsNullOrEmpty($_) }

14link}

Let’s use it to find the neighbours of some nodes within the random- and completeGraph we created earlier:

1linkGet-Neighbours $completeGraph.Edges 0

1linkGet-Neighbours $randomGraph.Edges 2

With the Get-Neighbours function, we can also create a function that iterates through a graph’s nodes connected by edges and return the nodes that can be reached from a given start node. This is implemented through Get-ReachableNodes, where we use a Stack to collect the neighbours and a HashSet to keep track of “visited” nodes.

1linkfunction Get-ReachableNodes ($Edges, $StartNode, [switch]$Undirected) {

2link    $seen = New-Object System.Collections.Generic.HashSet[string]

3link    $stack = New-Object System.Collections.Generic.Stack[string]

4link    $null = $stack.Push($StartNode)

5link    while ($stack.Count -gt 0) {

6link        $node = $stack.Pop()

7link        if (-not $seen.Contains($node)) {

8link            $null = $seen.Add($node)

9link            Get-Neighbours $Edges $node -Undirected:$Undirected | ForEach-Object {

10link                $null = $stack.Push( $_ )

11link            }

12link        }

13link    }

14link    return $seen

15link}

The last piece of the puzzle to check whether a graph is connected, is to use the Get-ReachableNodes function to get the set of nodes that can be reached from a giving starting point. If this number equals the number of nodes in the graph, that means we can reach all nodes, which means the graph is connected.

1linkfunction Get-IsConnected($Graph, [switch]$Undirected) {

2link    if ($Graph.Edges.Count -eq 0) { return $false }

3link    $startNode = $Graph.Edges[0].From

4link    $reachable = Get-ReachableNodes $Graph.Edges $startNode -Undirected:$Undirected

5link    $reachable.Count -eq $Graph.Nodes.Count

6link}

Our complete graph is connected and our random graph happened to be not connected:

1linkGet-IsConnected $completeGraph -Undirected

1linkGet-IsConnected $randomGraph -Undirected

linkProbability of connectivity

For given values, number of nodes, and probability, we would like to know the probability of the graph being connected. We can estimate this by generating a number of random graphs and counting how many are connected.

1linkfunction Get-ProbabilityConnected($NodeCount, $Probability, $Iterations = 100) {

2link    $count = 0

3link    for ($i = 0; $i -le $Iterations; $i++) {

4link        $randomGraph = Get-RandomGraph $NodeCount $Probability -Undirected -NoVisual

5link        if ((Get-IsConnected $randomGraph -Undirected )) { $count++ }

6link    }

7link    $count / $Iterations

8link}

1link$nodeCount = 10

2linkGet-ProbabilityConnected $nodeCount .23 100

23% was chosen because it is close to the critical value where the probability of connectivity goes from near 0 to near 1. According to Erdös and Renyi.

1link[System.Math]::Log($nodeCount) / $nodeCount

We can try to replicate the transition by estimating the probability of connectivity for a range of values of probabilities. We implement Python’s numpy.Logspace function to get an evenly spread list of probabilities within a range on a log scale:

1linkfunction Get-LogSpace([Double]$Minimum, [Double]$Maximum, $Count) {

2link    $increment = ($Maximum - $Minimum) / ($Count - 1)

3link    for ( $i = 0; $i -lt $Count; $i++ ) {

4link        [Math]::Pow( 10, ($Minimum + $increment * $i))

5link    }

6link}
